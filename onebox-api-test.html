ONEBOX API reference

The three calls used by the «Avísame» fan-alert flow: obtain a bearer token, list the sessions on our channel, and check seat availability per stadium sector for a given session. All examples are real captured exchanges with secrets stripped.

POST /oauth/token

Authentication

Exchanges the channel's client credentials for a short-lived bearer token (12-hour expiry). The token is sent as Authorization: Bearer … on every subsequent call.

POST /oauth/token HTTP/1.1
Host: api.oneboxtds.net
Content-Type: application/x-www-form-urlencoded
Accept: application/json

GET /catalog-api/v1/sessions

List sessions

Returns every session (match, event slot) currently bound to the authenticated channel. The «Avísame» backend polls this list to detect new sessions and to refresh metadata. Each item carries its event, venue, dates, sale window, and a top-level availability summary.

RequestBearer auth
GET /catalog-api/v1/sessions HTTP/1.1
Host: api.oneboxtds.net
Authorization: Bearer ••••••••••••••••••••••••••••••••
Accept: application/json
Response200 OK
Content-Type: application/json

GET /catalog-api/v1/sessions/{sessionId}/availability

Session availability

Per-session seat availability broken down by price type (PISTA, VIP, PISO 1–3) and by sector (the stadium grada). The «Avísame» rule engine watches this endpoint to decide when to fire fan alerts — e.g. "Pista Alta has just opened up".
